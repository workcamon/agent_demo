# YouTube 재생목록 매니저 (PWA)

유튜브 영상을 **여러 개의 재생목록(사용자 정의)**에 저장하고, **태그로 빠르게 검색**하는 PWA입니다.

## 기능

- **재생목록 여러 개**: 생성/이름 변경/삭제
- **영상 즐겨찾기 추가**:
  - URL 붙여넣기(데스크톱/모바일)
  - **모바일(설치형 PWA)**: 유튜브에서 `공유 → 이 앱`으로 바로 추가(Share Target)
  - **데스크톱**: “북마클릿”으로 재생 중 버튼처럼 사용(확장 설치 없이 가능)
- **태그**: 영상별 태그 입력/편집, `#태그` 검색, 태그 칩 클릭으로 검색어에 추가
- **오프라인/백업**: 로컬 저장 + JSON 내보내기/가져오기
- **주소로 공유**: 재생목록/태그/영상 정보를 **압축해서 링크 하나로 공유**(링크 열면 가져오기)

## 로컬 실행

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:5173` 로 접속하세요.

## 빌드

```bash
npm run build
npm run preview
```

## “유튜브 재생 중 버튼을 눌러 추가”에 대해

PWA는 **유튜브 페이지 내부에 버튼을 직접 삽입**할 수 없습니다(브라우저 확장/유저스크립트 영역).
대신 아래 2가지가 현실적인 대안입니다.

- **모바일(권장)**: PWA 설치 후, 유튜브에서 `공유 → 이 앱`으로 추가
- **데스크톱**: 앱의 “북마클릿” 기능으로 생성한 북마크를 누르면, 현재 재생 중 URL이 앱에 자동 입력됨

## 배포 팁(SPA 라우팅)

이 앱은 SPA라서 정적 호스팅에서는 `/share` 같은 경로가 **항상 `index.html`로 리라이트**되도록 설정해야 합니다.
예: Netlify/Vercel/Cloudflare Pages 등에서 “SPA fallback” 옵션을 켜면 됩니다.

## 링크 공유(압축 인코딩) 참고

PlantUML처럼 **텍스트(JSON) → deflate(raw) 압축 → base64url 인코딩**을 사용합니다.
링크가 너무 길어지면 메신저에서 잘릴 수 있으니, 그 경우에는 JSON 내보내기를 권장합니다.
